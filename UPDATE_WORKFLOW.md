# 🔄 Workflow обновления проекта

## Как работать с двумя копиями

```
┌──────────────────────────────────────────────────────────────┐
│  ЛОКАЛЬНАЯ КОПИЯ (у тебя на компе)                          │
│  C:\Users\asnov\...\Музыкальный агент                       │
│                                                              │
│  → Разработка                                               │
│  → Тестирование                                             │
│  → Исправления                                              │
└──────────────────────┬───────────────────────────────────────┘
                       │
                       │ git push
                       ▼
┌──────────────────────────────────────────────────────────────┐
│  GITHUB (публичный)                                         │
│  github.com/Aleksei-grand/MyFlowMusic                       │
│                                                              │
│  → Публичный доступ                                         │
│  → Issues от пользователей                                  │
│  → Docker образы                                            │
│  → Releases                                                 │
└──────────────────────────────────────────────────────────────┘
```

## Процесс обновления

### 1. Работаешь локально (здесь)
```bash
# Делаешь изменения в коде
# Тестируешь
# Исправляешь баги
```

### 2. Отправляешь на GitHub
```bash
# Добавляешь изменения
git add .

# Коммит (описываешь что изменил)
git commit -m "Fix: исправил баг с путями в Windows

- Добавил нормализацию путей
- Исправил кодировку для русских названий"

# Отправляешь
git push origin main
```

### 3. Создаёшь новую версию (когда готово)
```bash
# Обновляешь версию в коде
# music_agent/__init__.py
__version__ = "0.2.1-alpha"

git add .
git commit -m "Bump version to 0.2.1-alpha"

# Создаёшь тег (версию)
git tag -a v0.2.1-alpha -m "MyFlowMusic v0.2.1-alpha

Исправления:
- Фикс путей в Windows
- Улучшена обработка ошибок
- Добавлены тесты"

# Отправляешь тег
git push origin v0.2.1-alpha
```

### 4. GitHub Actions автоматически:
- Собирает Docker образ
- Публикует в GitHub Container Registry
- Создаёт Release (ты допишешь описание)

### 5. Пользователи получают:
- Уведомление о новой версии (если Watch → Releases)
- Новый Docker образ
- Обновлённый код

## Команды для быстрого обновления

```bash
# Быстрое обновление (одной командой)
git add . && git commit -m "Update: описание" && git push origin main

# Новая версия
git add . && git commit -m "Bump version" && git push origin main && git tag -a v0.2.1-alpha -m "v0.2.1-alpha" && git push origin v0.2.1-alpha
```

## Шаблоны коммитов

```bash
# Фикс бага
Fix: исправлено падение при пустом названии трека

# Новая фича
Feat: добавлена поддержка DistroKid

# Документация
Docs: обновлено руководство по установке

# Рефакторинг
Refactor: переписан модуль обработки аудио

# Тесты
Test: добавлены тесты для безопасности
```

## Обработка Issues от пользователей

1. **Пользователь создаёт Issue на GitHub**
2. **Ты читаешь здесь (локально)**
3. **Исправляешь баг здесь**
4. **Отправляешь на GitHub**
5. **Закрываешь Issue с комментарием**

```bash
# В коммите можно закрыть Issue
git commit -m "Fix: исправлено падение при синхронизации

Closes #3"
```

## Периодичность обновлений

| Тип | Период | Пример |
|-----|--------|--------|
| Хотфикс | Сразу | Критический баг |
| Минорный фикс | 1-2 раза в неделю | Мелкие баги |
| Фича | 1-2 раза в месяц | Новый функционал |
| Мажорный релиз | Раз в 3-6 месяцев | v0.3.0, v1.0.0 |

## Чеклист перед каждым push

- [ ] Код работает локально
- [ ] Тесты проходят (`pytest`)
- [ ] Нет секретов в коде (проверь `.env` не запушен)
- [ ] README обновлён (если нужно)
- [ ] CHANGELOG обновлён

## Если что-то пошло не так

### Откат изменений
```bash
# Откатить последний коммит (локально)
git reset --soft HEAD~1

# Откатить последний push (осторожно!)
git revert HEAD
```

### Исправить конфликт
```bash
# Стянуть изменения с GitHub
git pull origin main

# Решить конфликты в файлах
# Закоммитить результат
git add . && git commit -m "Merge: resolved conflicts"
git push origin main
```

**Запомни главное:**
- Работаешь здесь → Отправляешь на GitHub → Пользователи получают
- Коммиты часто → Тесты → Push → Готово!
